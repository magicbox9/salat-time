# Salat Time — macOS menu-bar client for Mawaqit

A native macOS menu-bar application that displays prayer times from any
[mawaqit.net](https://mawaqit.net) mosque, plays the adhan at prayer time,
and shows a live countdown in the menu bar.

Built as a companion client to the Mawaqit web platform — everything it
displays comes from Mawaqit's public API. This codebase is offered to the
Mawaqit team to use, modify, rebrand, host, or discard as they see fit,
with no strings attached.

## What it does

- Live prayer-times from any mosque on mawaqit.net (same schedules as the
  web site, fetched from `/api/2.0/mosque/search` and the per-mosque JSON)
- Menu-bar countdown to the next prayer, with a red highlight during the
  adhan-to-iqama window
- Plays the adhan at the scheduled time — 14 curated recordings bundled
  (Makkah, Madinah, Al-Aqsa, plus 11 other reciters sourced from Assabile)
- Configurable heads-up notifications N minutes before each prayer
- Country → city → mosque picker with live search, plus a distance-sorted
  Nearby tab using CoreLocation
- Full i18n in 10 languages: EN, FR, AR, ES, ZH, HI, BN, RU, PT, UR.
  RTL layout + Cairo font for Arabic
- Light / Dark / Auto appearance, Liquid Glass material support, and
  six accent-colour themes
- Per-prayer adhan mode toggles (off / notification-only / full adhan)
- Open at login, Hijri date display, 12h/24h toggle, configurable audio
  output device for the adhan

## Technology

- Swift + AppKit + WebKit + AVFoundation + CoreLocation + UserNotifications
- No Xcode project — the whole app is one Swift file compiled with a shell
  script. Deliberately kept this way so the codebase stays readable and
  contribution-friendly.
- No third-party Swift packages or CocoaPods. Zero runtime dependencies
  beyond the system frameworks.
- Target: macOS 14 (Sonoma) or newer.

## File layout

```
AwqatSalat/
├── main.swift         All app source (~4,900 lines)
├── build.sh           Compiles, downloads adhans + fonts, codesigns
├── Info.plist         Bundle metadata (version, bundle ID, LSUIElement)
├── icon.png           1024×1024 source icon
├── make_icon.py       Regenerates icon.png (crescent + stars)
└── Salat Time.app     Compiled output (regenerated by build.sh)
```

## Building from source

Requirements: macOS 14+, Xcode Command Line Tools (`xcode-select --install`).

```bash
./build.sh
open "Salat Time.app"
```

The build script will:
1. Compile `main.swift` into `Salat Time.app/Contents/MacOS/AwqatSalat`
2. Rasterize `icon.png` into a multi-resolution `AppIcon.icns`
3. Download the 14 adhan recordings from Assabile with a Referer header
4. Download the Cairo variable font from Google Fonts
5. Write `translations.json` for all 10 languages
6. Copy `Info.plist` into the bundle and ad-hoc codesign

Total build time: ~60 seconds on a first run (mostly the audio downloads).

## Data sources

- **Prayer times**: Mawaqit public API (`https://mawaqit.net/api/2.0/...`)
- **Mosque search / geocoding**: Mawaqit `/api/2.0/mosque/search?word=...`
- **Adhan audio**: [assabile.com](https://www.assabile.com) — 14 recordings
  from their curated public collection, downloaded at build time
- **Arabic font**: Google Fonts Cairo variable

No user accounts, no analytics, no telemetry, no network calls beyond the
mawaqit.net API.

## Known limitations

- Ad-hoc signed only. To distribute to end users without the Gatekeeper
  warning on first launch, the app needs to be signed with an Apple
  Developer ID certificate and notarized. Mawaqit presumably already has
  a Developer ID for the iOS / Android builds.
- Some mosques return incomplete addresses from the API; the app
  regex-extracts the city when the `city` field comes back empty
  (works for Morocco, France, Algeria; may miss edge cases elsewhere).
- The 14 adhans are downloaded at build time from Assabile's public CDN
  using a Referer header. If Assabile changes their CDN or hotlink
  policy, the download list in `build.sh` may need to be updated.
- Widget extension was removed from the codebase to keep things simple.
  Adding one back would be straightforward.
