# Handoff notes — Salat Time for the Mawaqit team

This document is written for the engineer on the Mawaqit team who will
first open the repo. The goal: zero to a running build in under 5 minutes.

## TL;DR

```bash
git clone <this-repo-url> salat-time
cd salat-time
./build.sh
open "Salat Time.app"
```

That's it. No Xcode project, no CocoaPods, no Swift package manifest, no
hidden state. Everything is one Swift file plus one build script.

## What's in each file

| File              | What it is                                                |
|-------------------|-----------------------------------------------------------|
| `main.swift`      | All app source (~4,900 lines, one file on purpose)        |
| `build.sh`        | Compile + download adhans/fonts + codesign in one pass    |
| `Info.plist`      | Bundle metadata (bundle ID, version, LSUIElement=true)    |
| `icon.png`        | 1024×1024 source icon, regenerated by `make_icon.py`      |
| `make_icon.py`    | PIL script that redraws `icon.png` (crescent + stars)     |
| `README.md`       | Feature overview + build instructions                     |
| `LICENSE`         | MIT, plus notes on third-party assets                     |
| `HANDOFF.md`      | This file                                                 |

Everything else in the tree (`Salat Time.app/`, `AppIcon.iconset/`,
downloaded fonts + adhans) is regenerated by `build.sh` and is gitignored.

## Dependencies

- macOS 14 (Sonoma) or newer, target is macOS 14+
- Xcode Command Line Tools: `xcode-select --install`
- Python 3 with Pillow if you want to regenerate `icon.png`:
  `pip3 install Pillow`
- A network connection the first time you build (for fonts + adhans)
- No Swift packages, no CocoaPods, no Homebrew, no Node

## Build flow (what `./build.sh` does)

1. `swiftc main.swift …` → `Salat Time.app/Contents/MacOS/AwqatSalat`
2. Rasterizes `icon.png` at 10 sizes, bundles into `AppIcon.icns`
3. Downloads the 14 adhan MP3s from `media.assabile.com` (with Referer)
4. Downloads the Cairo variable font from Google Fonts
5. Writes `translations.json` (10 languages, inlined in the script)
6. Copies `Info.plist`, ad-hoc codesigns the bundle

Typical build time: ~60 seconds on a first run (downloads dominate),
~5 seconds on subsequent runs (downloads cached in the Resources dir of
the previous build; delete the .app to force re-download).

## Rebranding checklist

If the Mawaqit team wants to ship this as an official Mawaqit product,
these are the handful of touch-points to change:

1. **Bundle ID** — `Info.plist` → `CFBundleIdentifier`
   (currently `com.oumsai.awqatsalat` — change to `net.mawaqit.salattime`
   or whatever ID matches your Developer account)
2. **App name** — `build.sh` → `APP_NAME="Salat Time"` if you want a
   different display name
3. **Binary name** — `build.sh` → `BINARY="AwqatSalat"` and the
   `CFBundleExecutable` in `Info.plist` (keep them in sync)
4. **Code signing** — replace the ad-hoc codesign line at the end of
   `build.sh` with your Developer ID signature and add a notarization
   step. Roughly:
   ```bash
   codesign --force --deep --options runtime \
       --sign "Developer ID Application: Mawaqit (TEAMID)" \
       --entitlements entitlements.plist \
       "$APP_DIR"
   xcrun notarytool submit "$APP_DIR.zip" \
       --keychain-profile "AC_NOTARY" --wait
   xcrun stapler staple "$APP_DIR"
   ```
5. **Icon** — either keep the crescent+stars or drop in your own 1024×1024
   PNG at `icon.png` and run `./build.sh` again
6. **Accent palette** — `main.swift` → search for `kAccentPalette` if you
   want to replace the six theme colors with Mawaqit brand colors

## Data sources — what calls what

- **Prayer times**: `mawaqit.net/api/2.0/mosque/<slug>` (JSON, no auth)
- **Mosque search**: `mawaqit.net/api/2.0/mosque/search?word=<q>` (JSON)
- **Geolocation**: `mawaqit.net/api/2.0/mosque/geolocation/<lat>/<lon>`
- **Adhan audio**: `media.assabile.com/…/*.mp3` (build time only)
- **Cairo font**: `fonts.google.com` (build time only)

No third-party tracking, analytics, or cloud storage. All user prefs
live in `~/Library/Preferences/com.oumsai.awqatsalat.plist` (renamed
once you change the bundle ID).

## Architecture at a glance

`main.swift` has three main areas, each marked with a `// MARK:` banner:

1. **Globals & models** (top, ~lines 1–900) — constants, `MosqueInfo`
   codable struct, `AdhanOption` catalog, Localizer, helpers
2. **View controllers** (middle, ~lines 900–3400) — `MosquePickerView`,
   `SettingsView`, popup menus, swatch buttons
3. **AppDelegate** (bottom, ~lines 3400–4900) — status-bar wiring,
   popover, fetcher, adhan scheduler, notifications, countdown tick

Quickest way to find something: grep for the user-facing string in
`build.sh` (where `translations.json` lives), copy the key, grep for
`t("that.key")` in `main.swift`.

## Known limitations

- Ad-hoc signed. Users will see the Gatekeeper warning on first launch
  until you re-sign with a Developer ID.
- Some Mawaqit records return empty `city` — the app regex-extracts it
  from the `localisation` string. Works for Morocco / France / Algeria,
  may miss edge cases.
- `build.sh` assumes the Assabile CDN keeps the current URL scheme. If
  they change it, update the `download_adhan_assabile` URL list.
- No automated tests. The codebase is small enough that manual QA on a
  handful of mosques has been sufficient so far.
